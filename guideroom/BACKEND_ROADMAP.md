# Backend & Infrastructure Guide (백엔드 입문 가이드)

웹사이트에 "생명력"을 불어넣는 백엔드와 이를 세상에 공개하는 인프라에 대해 알아봅니다.

## 1. 백엔드의 핵심 역할
- **데이터 보관(DB)**: 누가 상담 신청을 했는지, 현재 에너지 단가는 얼마인지 등을 저장합니다.
- **비즈니스 로직**: "상담 신청이 오면 사장님 폰으로 카톡을 보낸다" 같은 규칙을 실행합니다.
- **인증(Auth)**: 관리자만 접속할 수 있는 페이지를 보호합니다.

## 2. 어디에 서버를 두나요? (Infrastructure)

### A. Vercel (현재 방식)
- **특징**: 프론트엔드와 백엔드(Next.js API)를 동시에 해결합니다.
- **장점**: 가장 쉽고, 전 세계에서 가장 빠르며, 소규모 사업자에게는 거의 무료입니다.

### B. AWS (Amazon Web Services)
- **특징**: 전 세계 1위 클라우드입니다. 
- **용도**: 단순히 웹사이트를 넘어 대규모 앱, 인공지능 서버 등을 운영할 때 씁니다.
- **단점**: 설정이 매우 복잡하고, 실수하면 요금 폭탄이 나올 수 있어 초보에게는 주의가 필요합니다.

### C. Naver Cloud / 가비아 / 카페24
- **특징**: 한국 서비스입니다. 결제가 편하고 한국어 고객지원이 잘 됩니다.
- **용도**: 한국 내 데이터 규제가 엄격하거나, 도메인(.kr, .co.kr) 구매와 서버를 한 곳에서 관리하고 싶을 때 씁니다.

## 3. 유지보수(Maintenance)는 무엇을 하나요?
- **모니터링**: 서버가 죽지 않았는지 24시간 감시합니다.
- **백업**: DB를 매일 복사해두어 사고 시 복구합니다.
- **보안 업데이트**: 최신 해킹 기법에 대응하기 위해 라이브러리 버전을 올립니다.
